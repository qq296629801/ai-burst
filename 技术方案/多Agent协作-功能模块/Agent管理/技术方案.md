# Agent 管理（技术方案节选）

> 完整版：[多Agent协作技术方案.md](../../多Agent协作技术方案.md)

## 领域模型

- **§4.2 Agent 与沟通**：**`mag_agent`** — `project_id`、`role_type`（PM/PRODUCT/BACKEND/FRONTEND/TEST/VERIFY）、`parent_agent_id`（主/子）、`llm_channel_id`、`system_prompt_profile`、`extra_json`、`status`。

## AgentScope 与工具

- **§5.1** 与 `llm_channel` 的桥接；每行 `mag_agent` 可绑定通道。
- **§5.2 Agent 与工具**：职能 Agent / 核查 Agent 工具注册原则；**禁止自审**（`VERIFY` 不得为同任务 assignee）。
- **§5.3～5.4** 编排与 Temporal 分工、超时与费用 — 与运行时相关。

## REST API（摘录）

- `GET/POST /api/mag/projects/{id}/agents`、`PUT /api/mag/agents/{id}`  
- 可选 `POST /api/mag/threads/{id}/run` 或 `/agents/{id}/run`

## 权限

- **§10**：`mag:agent:manage`；默认矩阵见完整版 §10.1。

## 前端

- **§11**：工作台 Agent Tab；路由 `/mag/project/:id`。
