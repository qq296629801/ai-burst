# MAG 多 Agent 协作 — 功能使用指南（小白版）

本文面向**第一次使用** AI Burst 中「多 Agent 协作（MAG）」模块的用户，用**最少术语**说明：**从登录到日常操作**要点。更完整的产品能力说明见 [多Agent协作与项目管理产品线需求.md](./多Agent协作与项目管理产品线需求.md)。

---

## 1. 使用前准备

### 1.1 你需要有什么

- 已在浏览器打开系统地址（本地开发一般为 [http://localhost:5173](http://localhost:5173)，前端会把接口转到后端）。
- 后端服务已启动（默认 [http://localhost:8080](http://localhost:8080)），数据库与 Flyway 迁移正常（否则页面可能报错或无法登录）。
- 管理员已为你开通**账号**，并分配了 MAG 相关**权限**（见下文「为什么有的菜单/按钮没有」）。

### 1.2 默认管理员（仅本地/首次环境）

首次启动且无 `admin` 用户时，系统可能自动创建：

- **用户名**：`admin`
- **密码**：`admin123`

生产环境请务必修改密码与权限策略。升级数据库迁移（如新增 MAG 菜单）后，若看不到新菜单，请**重新登录**一次。

---

## 2. 登录与主界面

### 2.1 登录

1. 打开登录页，输入**用户名**和**密码**。
2. 点击登录。若失败，检查大小写、网络或联系管理员。
3. 登录成功后，会进入**主布局**：左侧是菜单，上方显示昵称/用户名，可点**退出**。

### 2.2 菜单从哪来

左侧菜单**不是**每个人都一样，由管理员在「角色 / 权限」里配置。MAG 相关常见菜单包括（名称以你环境为准）：

| 菜单大致名称 | 作用简述 |
|-------------|---------|
| **MAG 项目** | 项目列表；从这里进入某个项目的**工作台**。 |
| **MAG 大屏** | 项目维度的运行快照（任务统计等，随实现展示）。 |
| **MAG 待办** | 需要你**拍板**等事项的聚合列表（有权限才显示）。 |
| **MAG 知识库** | 组织知识条目的维护与检索入口。 |
| **MAG 归档** | 发版/发布版本类归档列表（有权限才显示）。 |

若整项菜单都没有：通常是**没有对应权限**，需联系管理员，不要反复刷新指望「刷出来」。

---

## 3. 推荐操作路径（日常最常用）

### 3.1 从「项目」到「工作台」

1. 在左侧点开 **MAG 项目**（或名称相近的「项目」菜单）。
2. 你会看到**项目列表**（名称、状态、Agent 数量、需求版本、最近活动等）。
3. 某一行的 **「工作台」** 按钮：点击进入该项目的 **MAG 工作台**（一个项目一个工作台页面）。

> **说明**：工作台地址形如 `/mag/project/项目ID`，请通过列表进入，不必手敲地址。

### 3.2 新建项目（有权限时）

1. 在 **MAG 项目** 列表页，若能看到 **「新建项目」**，点击后输入项目名称并保存。
2. 新建成功后，在列表里点 **「工作台」** 进入该项目继续配置。

---

## 4. 项目工作台里有什么（按标签页）

进入某个项目的 **工作台** 后，页面上方是项目标题与状态；下方是多个 **标签页**。你**能看到哪些标签**与**能否点某些按钮**，仍取决于权限。

下面按标签**从左到右的常见顺序**说明**可以做什么**（与当前前端实现对齐）。

### 4.1 成员

- 查看本项目已加入的**真人用户**（用户 ID、用户名、昵称、在项目内的角色等）。
- 添加/移除成员等能力若后续在界面提供，同样会受权限控制。

### 4.2 Agent

- 查看本项目下的 **Agent 列表**（名称、类型、状态等）。
- **新建 Agent / 编辑**（有 `mag:agent:manage` 等权限时）：选择类型、填写名称；**大模型通道** 下拉框绑定本人「通道配置」里的通道（`llmChannelId`）。**触发 run** 前须绑定，否则接口报错。
- **触发 run**（有任务操作类权限时）：**须先在 Agent 上绑定大模型通道**（`llmChannelId`，可通过更新 Agent 接口配置）；未绑定会直接 **接口报错**（业务码 `MAG_AGENT_LLM_CHANNEL_REQUIRED`），不会启动 Workflow。绑定后：若 `aiburst.mag.temporal.enabled=true` 且能连上 Temporal，会启动 Workflow（返回体可含 `workflowId`）；未启用或连不上则 **警告** 且 `accepted=false`。
- **编排执行**（项目页签）：展示每次 **Agent / 沟通线程** 触发 run 后的记录（状态：已提交 → 执行中 → 成功/失败，或未接受）；支持刷新，并可通过 **WebSocket**（订阅本项目）在 Worker 更新状态时自动刷新列表。

**小白提示**：Agent 可以理解成「不同角色的智能助手实例」，类型里常见后端、前端、测试、产品、项目经理等（与产品文档一致）。

### 4.3 模块

- 维护本项目的**功能模块**树（增删改）。
- **导入蓝图**：从知识库条目或归档等来源导入模块结构（有对应权限时）。

### 4.4 任务

- 查看任务列表与状态（如待处理、进行中、阻塞等）。
- **新建任务**、**开始**、**申报完成**、**阻塞**、**要活** 等按钮在状态与权限允许时出现。**申报完成**可手动点击；在派工自动编排等场景下，系统也可在满足规则时**自动**将任务标为已完成（与产品 §4.5、技术方案一致，可在后端配置关闭）。
- **要活**：子 Agent 侧常用语义，表示「申请下一项工作」，与产品里的协调流程一致。

### 4.5 需求文档

- 在中间大文本框里编辑需求正文，**保存新版本**会生成新版本记录。
- **版本列表**：查看历史版本。
- **对比**：选择两个版本号，查看差异结果（展示形式依后端返回而定）。
- **变更影响分析**：提交分析请求后，会返回追踪 ID 等（具体字段以后端为准）。

### 4.6 需求池

- 展示需要**真人参与决策**的条目（状态会标明是否待用户处理等）。
- 对待拍板项可进行：**原样通过 / 变更通过 / 拒绝 / 延期** 等（有拍板权限时）。
- **产品结案**：产品侧关闭条目并记录结论（有需求编辑类权限时）。

### 4.7 归档

- 查看与本项目相关的**发布/版本归档**记录。
- **新建归档**（有发版归档权限时）：按表单填写后保存。

### 4.8 沟通

- **新线程**：创建协作线程。
- **消息**：在线程里收发消息（对话框形式）。
- **触发编排**：对有权限用户提供的占位触发（与 Agent 编排链路相关）。

### 4.9 告警

- 查看项目告警列表，可对未确认项点 **确认**。

### 4.10 PM 协助

- 查看项目经理协助记录；有权限时可 **登记协助**。

### 4.11 改进

- 先选一个 **Agent**，再 **加载日志** 查看改进记录；有权限时可 **追加记录**。

### 4.12 定时任务（有权限才显示）

- 配置或更新 Cron 类定时任务；可刷新查看本项目相关任务。

## 5. MAG 项目外的独立页面

### 5.1 MAG 大屏

- 选择或输入**项目**，查看大屏快照（指标以后端返回为准）。
- 用于「一眼看项目整体情况」，不等同于工作台里的细项编辑。

### 5.2 MAG 待办

- 进入后分页查看与你相关的**待办/待拍板**类事项（与需求池等联动，具体列以后端为准）。

### 5.3 MAG 知识库

- 维护知识库条目：新建、编辑、删除、分页查询等（按钮随权限显示）。

### 5.4 MAG 归档

- 组织或列表视角的归档入口（与项目工作台内「归档」标签可能互补，以你环境菜单为准）。

## 6. 常见问题（小白向）

### 6.1 为什么我没有「新建项目」「新建 Agent」等按钮？

这些操作对应**独立权限**。联系管理员为你的角色勾选例如：项目管理、Agent 配置、任务与申报、需求文档、需求池拍板等权限后，**重新登录**再试。

### 6.2 打开页面提示 403 / 无权限

- 说明当前账号**无权访问该路由或操作**。
- 把**浏览器地址栏路径**和你要做的事告诉管理员，便于对照权限码调整。

### 6.3 点了「触发 run / 触发编排」好像没反应？

这类操作往往只是**把请求发给后端**；真正执行依赖 **Worker、Temporal、模型通道** 等是否已部署配置。开发环境可能仅返回「已接受」类结果，属正常现象，可让技术人员查日志。

### 6.4 登录后菜单是空的或很少

- 账号可能被分配了登录权限，但**没有菜单型权限**。
- 需要管理员在「角色 / 菜单权限」里分配 **MAG** 相关菜单。

### 6.5 接口报错、空白页

- 确认后端已启动、本机代理正常（前端开发时 `/api` 转发到后端）。
- 打开浏览器 **开发者工具 → 网络**，看红色失败的请求，把路径与状态码发给技术支持。
