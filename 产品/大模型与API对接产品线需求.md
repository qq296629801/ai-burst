# 大模型与 API 对接 — 产品线需求

## 1. 背景与目标

平台需在**统一体验**下对接多家大模型厂商与开放接口，使用户可配置自有 **API Key / Endpoint**，并在受控权限内完成**对话、评测与后续业务编排**（如 RAG、Agent 等可在此基础上扩展）。

**目标**

- 支持多厂商、多协议形态（以 **OpenAI 兼容**为主干，辅以原生协议适配）。
- 通道级配置（密钥、Base URL、默认模型）可管理、可审计。
- 调用可走**平台统一网关**（鉴权、限流、日志留痕在后续迭代强化）。

---

## 2. 支持厂商 / 模型生态（清单）

下列为产品需**覆盖规划**的厂商或系列（含用户点名的与补充的主流国内国际服务）。实际落地按**协议类型**归并实现，避免为每家重复造轮子。

| 类型 | 厂商 / 产品 | 说明 |
|------|-------------|------|
| 国际通用 | **OpenAI（ChatGPT）** | Chat Completions / Responses 等官方 API |
| 国际通用 | **Anthropic Claude** | Messages API（原生协议） |
| 国际通用 | **Google Gemini** | Generative Language API（原生协议，后续迭代） |
| 国际通用 | **Azure OpenAI** | 与 OpenAI 形态一致，Endpoint 与密钥体系不同 |
| 国际通用 | **Groq** | OpenAI 兼容 HTTP 接口 |
| 国产 / 国内云 | **阿里云通义（DashScope / Qwen）** | 兼容模式或 DashScope 原生 HTTP |
| 国产 / 国内云 | **深度求索 DeepSeek** | OpenAI 兼容接口为主 |
| 国产 / 国内云 | **智谱 GLM（ChatGLM）** | 提供 OpenAI 兼容与自有 API |
| 国产 / 国内云 | **MiniMax** | 官方提供 OpenAI 兼容与自有接口 |
| 国产 / 国内云 | **月之暗面 Moonshot（Kimi）** | OpenAI 兼容 |
| 国产 / 国内云 | **百川 Baichuan** | OpenAI 兼容或自有接口 |
| 国产 / 国内云 | **字节豆包（Volcengine）** | 方舟等 OpenAI 兼容或自有接口 |
| 国产 / 国内云 | **腾讯混元** | 自有 / 兼容形态以官方为准 |
| 国产 / 国内云 | **百度千帆 / 文心** | 自有接口（后续适配） |
| 工具链 / IDE | **Cursor** | 无独立「大模型对话」公开 API；产品侧定义为 **OpenAI 兼容客户端可指向的代理/兼容端点**（用户自建网关或合规代理），不承诺逆向非公开接口 |
| 私有化 / 本地 | **Ollama、vLLM、LocalAI 等** | OpenAI 兼容本地 Base URL |

**原则**：凡官方提供 **OpenAI 兼容 Base URL** 的，一律走统一 **OpenAI Compatible 适配器**；仅兼容无法覆盖的再走 **专用适配器**（如 Claude、DashScope 原生）。

## 4. 功能需求

### 4.1 厂商与协议元数据

- 系统内置**厂商字典**：编码、展示名、默认 Base URL 提示、推荐协议类型（OPENAI_COMPAT / ANTHROPIC / DASHSCOPE / GEMINI…）。
- 字典可随版本发布更新；**不在此表内的厂商**允许用户**自定义 provider 编码 + 自定义 Base URL**（高级能力，可选首期简化为只选内置编码）。

### 4.2 通道（Channel）管理

- 通道字段至少包含：名称、`provider_code`、**Base URL**、**API Key（加密存储）**、默认模型、扩展 JSON（如 deployment、region）、启用状态。
- 通道归属：**创建人（owner）**维度管理；仅本人或管理员可改删（权限点控制）。
- Key **永不明文回显**；保存后仅支持「重新填写覆盖」。

### 4.3 统一对话调用（首期）

- 入参：通道 ID、消息列表（role + content）、可选模型覆盖、可选温度等基础参数。
- 出参：统一结构：`content`、`model`、`usage`（可空）、`rawProvider`（可选调试）。
- **流式输出（SSE）**作为二期；首期 **同步 JSON** 即可。

### 4.4 权限与安全

- 菜单：大模型目录下「通道配置」「对话体验」。
- 接口：`llm:channel:*`、`llm:chat:invoke` 等与 RBAC 对齐。
- 传输：**HTTPS**；日志**脱敏**（不打印完整 Key）。

### 4.5 非功能（首期简版）

- 超时：可配置，默认如 60s。
- 失败提示：区分网络错误、4xx/5xx、厂商返回体解析失败。
- 后续：限流、熔断、缓存、计费、多租户隔离。

---

## 5. 验收要点（摘要）

- 内置字典包含本节表格中的主流厂商编码，且前端可选。
- 用户可新增一条 OpenAI 兼容通道（如 DeepSeek 官方兼容地址），在对话页能返回模型回复。
- Claude / 通义等：至少一种走专用适配器可打通；其余可文档标明「兼容模式填同一适配器」。
- 无权限用户无法调用 `chat` 与通道 CRUD。
