# 导读与范围（节选）

> 完整版：[多Agent协作与项目管理产品线需求.md](../../多Agent协作与项目管理产品线需求.md)

## 1. 背景与目标

在 AI Burst 平台内，用户需要以**项目**为单位组织研发活动：通过**可配置的 Agent** 模拟或辅助真实团队角色（后端、前端、测试、产品、项目经理等），使多个 Agent **分工协作、可观测地沟通**，并在**需求变更**时驱动全链路任务与交付物对齐。

**产品目标**（与全量模块相关，完整列表见主文档 §1）包括：Agent 录入与协同、项目工作台、主/子层级与派工、全量行为与改进轨迹、定时任务、需求文档驱动变更、运营大屏与告警、阻塞升级与窄口径真人拍板、发版归档与项目隔离、组织知识库、待用户拍板入口、**核查 Agent 作为任务完成的准绳**等。

---

## 2. 范围

**交付策略**：主文档 **§2.1 所列模块均为本期建设范围**，以**全量交付**为目标；实施顺序由项目计划排期，**能力清单不缩减**。

### 2.2 本期不包含或后续迭代（摘要）

- 与真实 IM（企业微信、Slack）的双向深度集成（可先导出/ webhook）。
- 自动代码合并到生产环境（可与 CI 对接作为扩展）。
- 跨租户项目协作（多租户隔离策略另立需求）。
