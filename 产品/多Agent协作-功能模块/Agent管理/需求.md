## §3 Agent 类型与职能（预设角色）

下列为产品需内置或可映射的**角色类型**；每类 Agent 具备**独立职能说明**与**默认可执行动作范围**（具体工具权限由技术方案约束）。

| 类型 | 职能概要 |
|------|----------|
| **项目经理（PM）Agent** | 维护项目里程碑与模块划分；接收各主 Agent 的资源与进度诉求；按功能模块与优先级**派工**；**受理执行 Agent 的阻塞说明并协助排障**；无法协调时**升级至产品侧**；汇总风险与变更影响 |
| **产品 Agent** | 维护需求文档结构、用户故事与验收标准；发起需求澄清；在变更时标注**受影响范围**；在升级链路中**优先查组织知识库**、再**检索已落地成熟产品**；**多款成熟产品做法并存时须对比择优**并留痕；**仅当知识库无据且检索仍无合格产品命中**时生成**待用户拍板**项 |
| **后端开发 Agent** | API、数据模型、服务实现相关任务；可挂载**子 Agent** 分担接口/模块 |
| **前端开发 Agent** | 页面、组件、交互与联调相关任务；可挂载子 Agent 分担页面/模块 |
| **测试 Agent** | 用例、自动化、缺陷跟踪与回归策略；可挂载子 Agent 分担模块或类型 |
| **核查 Agent** | **独立**于执行方：当**任意职能 Agent**申报任务完成时进行核验；可**检索**组织知识库、外部已落地成熟实践；**仅核查 Agent 判定「通过」后**任务方可进入**已完成**终态；**每一次通过/不通过判定均须落库留存**（见 **§4.5**） |

*允许扩展更多角色类型；**六类** Agent 角色为**必选基线**。测试 Agent 与核查 Agent 职责不互相替代。*

---

## §4.5 中与「核查 Agent 配置」相关的概要（交叉引用）

- 项目内须启用至少 **1 个核查 Agent 实例**（或与任务类型绑定的多实例策略由技术方案定义）；核查策略模板可按职能线区分，但**角色类型**统一为 **核查 Agent**。

完整 §4.5 见 [任务完成核查](../任务完成核查/需求.md)。
