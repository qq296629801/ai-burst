# 协调链 + 任务与申领（节选）

> 完整版：[多Agent协作与项目管理产品线需求.md](../../多Agent协作与项目管理产品线需求.md)

## §2.1 模块说明

| 模块 | 说明 |
|------|------|
| 协调链 | 项目经理 Agent 协调各职能主 Agent；主 Agent 协调本子职能子 Agent |
| 任务与申领 | 子 Agent 完工后发起「要活」；主 Agent 向项目经理同步缺口；项目经理按模块与进度派工 |

---

## §4.1 层级模型

- 每个职能线（如后端）可配置 **1 个主 Agent + N 个子 Agent**。
- **主 Agent** 负责：拆解本子线任务、分配给子 Agent、合并产出、向 **项目经理 Agent** 汇报进度与资源需求。
- **子 Agent** 负责：执行原子任务；**任务完成后主动请求主 Agent 分配下一项工作**（「要活」）。

---

## §4.2 与项目经理的协作

- 主 Agent 在子线任务饱和或枯竭时，向 **项目经理 Agent** 请求新任务或调整优先级。
- **项目经理 Agent** 依据：当前**项目开发进度**、**功能模块清单**、依赖关系与里程碑，进行**分工与再平衡**。
- 用户可在界面侧看到：待派发队列、进行中、阻塞原因（若有）。

---

## §5.2 任务与模块（与任务状态、核查衔接部分见任务完成核查目录）

- 功能模块与任务建议支持树或标签，与需求条目可**双向关联**。
- 任务状态须支持 **§4.5** 闭环，至少包含：**待派发**、**进行中**、**待核查**、**核查中**、**已完成**（**仅**在核查 Agent **核查通过**后进入）、**阻塞**；**核查不通过**时任务须**退回进行中**并保留核查意见。阻塞需记录原因与责任人 Agent。

完整任务状态与核查规则见 [任务完成核查](../任务完成核查/需求.md)。

---

## §9 验收要点（与本模块相关）

- 子 Agent 完成工作后能触发 **「要活」流程**，主 Agent 向项目经理 Agent **请求派工**，项目经理按 **模块与进度** 更新任务分配。
