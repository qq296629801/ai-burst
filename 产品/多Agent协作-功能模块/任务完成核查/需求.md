# 任务完成核查（节选）

> 完整版：[多Agent协作与项目管理产品线需求.md](../../多Agent协作与项目管理产品线需求.md)

## §2.1 模块说明

| 模块 | 说明 |
|------|------|
| 任务完成核查 | **核查 Agent** 对各职能 Agent **申报完成**的任务独立核验（含检索组织知识库与外部成熟实践以充分验证）；**非执行方自述完成即完成**（见 §4.5） |

---

## §4.5 任务完成申报与核查 Agent（必选）

**原则**

- **任意职能 Agent**在系统中标记或声明**「任务已完成」**时，**不得**直接将任务状态置为最终 **已完成**。
- **核查 Agent** 必须介入：对照该任务关联的**需求条目、验收标准、约定产出物**，判断执行方主张是否成立。
- **充分验证**：核查 Agent 应**主动检索**（组织知识库、与任务类型相关的**已落地成熟产品**实践、公开技术方案等）以**比对**完成度；必要时要求执行方**补充证据**。

**结论与权威**

- **通过**：任务进入 **已完成** 终态，方可参与统计「真正完成」、大屏效率与发版归档口径。
- **不通过**：任务**退回**执行方（通常回到 **进行中**），整改后可再次申报完成并重新进入核查。
- **是否完成以核查 Agent 结论为准**；执行方申报仅表示**待核查**。

**核查记录留存（必录）**

- **每一次判定**（**通过**与**不通过**）均须**持久化留存**；**通过/不通过**记录字段要求见主文档 §4.5；同一任务多次核查须**保留历次记录**。

**与测试 Agent 的关系**

- **测试 Agent** 承担的任务申报完成时，**同样**须经 **核查 Agent** 核验。
- **产品 Agent、项目经理 Agent** 所承担的任务申报完成时，**同样**必须经 **核查 Agent** 核验。

**配置（概要）**

- 项目内须启用至少 **1 个核查 Agent 实例**；核查策略模板可按职能线区分。

---

## §5.2 任务与模块（状态机与本模块直接相关部分）

- 任务状态至少包含：**待派发**、**进行中**、**待核查**、**核查中**、**已完成**（**仅**核查 **通过** 后）、**阻塞**；原「待验收」语义由 **待核查 / 核查中** 承接；**不得弱化**「非核查 Agent 通过不得标已完成」的硬规则。

---

## §9 验收要点（与本模块相关）

- 任意职能 Agent 申报任务完成后，任务进入 **待核查/核查中**，由 **核查 Agent** 检索与验证；**核查通过**后才可进入 **已完成**；**每一次核查通过/不通过**均在系统内**可查、可追溯**。
