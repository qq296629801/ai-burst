# 阻塞与升级链路（节选）

> 完整版：[多Agent协作与项目管理产品线需求.md](../../多Agent协作与项目管理产品线需求.md)

## §2.1 模块说明

| 模块 | 说明 |
|------|------|
| 阻塞与升级链路 | 派工后无法执行时的主动沟通；项目经理协助与记录；升级至产品侧；知识库 + **检索成熟产品**、**多方案对比择优**；**仅知识库无据且检索搜不到成熟产品**时进入待用户拍板 |

---

## §1 产品目标（相关条）

- 派工后**无法落地**时，执行 Agent 须**主动向项目经理 Agent 说明原因**；项目经理**协助排障**并**留痕**；无法协调时升级至**产品侧**研判；**全程可审计**。

---

## §4.3 派工后无法执行时的沟通（必选）

- 任意 Agent 领到任务后若**无法开始或无法完成**，须**主动发起**与项目经理 Agent 的协作消息，说明**干不了的原因**。
- **项目经理 Agent** 须响应并尝试协调；**每一次协助**须写入**项目经理协助记录**（见 §5.6）。

---

## §5.6 升级链路、项目经理协助记录与产品—需求池

**产品线 Agent 主导、人工拍板窄口径**、**5.6.1 成熟产品闭环门槛**、**升级顺序（执行链）**、**项目经理协助记录（必录）**、**记录与可见性** — 全文见主文档 §5.6 与 §5.6.1。

与**需求池**状态、**待用户拍板**卡片规则衔接见 [需求池](../需求池/需求.md)。

---

## §9 验收要点（与本模块相关）

- 派工后无法执行的 Agent 能**主动向项目经理说明原因**；可查**项目经理协助了哪些问题、哪些 Agent**；升级至产品后，成熟产品命中与多竞品对比择优留痕符合 **5.6.1**；**仅窄口径**产生**待用户拍板**。
