# 权限产品线需求

## 1. 背景与目标

AI Burst 为前后端分离架构，需在平台层提供**统一身份认证与授权**，使各业务模块共享同一套用户、角色与权限数据，避免重复造轮子与安全口径不一致。

**产品目标**

- 管理员可配置组织内的用户、角色、菜单与接口权限。
- 终端用户登录后仅能看到被授权的菜单与操作，且后端拒绝未授权请求。
- 权限变更在合理时间内对用户侧生效（见非功能要求）。

---

## 2. 范围

### 2.1 本期包含

| 模块 | 说明 |
|------|------|
| 登录 / 登出 | 账号密码登录（首期）；登出使令牌失效或会话结束 |
| 用户管理 | 用户 CRUD、启用/禁用、重置密码、分配角色 |
| 角色管理 | 角色 CRUD、为角色绑定菜单与接口权限 |
| 菜单与权限资源 | 树形菜单；权限标识与前端路由、后端 API 对齐 |
| 个人中心 | 查看/修改基础资料、修改密码（可选首期） |
| 操作审计（建议） | 关键管理操作留痕（谁在何时改了什么） |

### 2.2 本期不包含（可列入路线图）

- 多租户（SaaS 级数据隔离）
- SSO / OAuth2 / 企业微信等第三方登录
- 数据权限（按部门/本人可见行级规则）
- 双因素认证（2FA）

---

## 3. 角色与用户故事

### 3.1 角色定义

| 角色 | 描述 |
|------|------|
| 系统管理员 | 维护用户、角色、菜单/权限全量配置 |
| 业务管理员 | 仅管理本部门或子集用户（若后续扩展组织树） |
| 普通用户 | 使用已授权功能，无配置权限 |

*首期可只实现「系统管理员」与「普通用户」，业务管理员按迭代增加。*

### 3.2 用户故事（示例）

1. **作为系统管理员**，我希望新增用户并分配角色，以便新员工能登录且只能访问其职级对应功能。
2. **作为系统管理员**，我希望新增角色并勾选菜单与接口权限，以便快速复制常见岗位权限模板。
3. **作为普通用户**，我希望登录后侧边栏只显示我有权访问的菜单，避免误点无权限页面。
4. **作为安全负责人**，我希望未登录访问受保护接口时返回统一未认证错误，已登录但无权限时返回无权限错误，且不泄露系统内部信息。

## 4. 功能需求

### 4.1 认证

- 支持用户名 + 密码登录；密码存储使用单向哈希（具体算法由技术方案约定）。
- 登录成功颁发访问凭证（如 JWT 或 Session），前端持久化策略需与安全要求一致。
- 支持登出：服务端使凭证失效或清除会话（若用 JWT，需黑名单或短有效期 + 刷新策略）。
- 连续登录失败可锁定账号或可配置策略（可选，首期可简化）。

### 4.2 用户管理

- 列表分页、按用户名/状态筛选。
- 新增/编辑：用户名唯一、必填项校验；可选手机号/邮箱等扩展字段。
- 启用/禁用：禁用用户不可登录。
- 分配角色：多选角色；保存后立即影响该用户后续请求授权结果（结合缓存策略见技术方案）。

### 4.3 角色与权限

- 角色名称/编码唯一；支持描述字段。
- 权限配置维度建议至少包含：**菜单（含路由 meta）**、**按钮/操作**、**后端 API（权限码）**；首期可菜单 + API 合一用统一 permission 编码，由前后端约定前缀区分。
- 支持按树勾选菜单；接口权限可与菜单关联或独立列表勾选（产品需二选一或并存，建议并存以便开放 API 场景）。

### 4.4 前端表现

- 动态路由：根据登录后拉取的菜单/权限生成可访问路由表。
- 菜单隐藏无权限项；按钮级无权限则隐藏或禁用（产品统一为一种交互，建议无权限则隐藏）。
- 直接输入 URL 访问无权限路由时跳转 403 或提示页。

### 4.5 后端表现

- 所有业务接口（除登录、验证码、静态配置白名单外）需认证。
- 管理类接口需额外角色或权限点校验。
- 返回 HTTP 状态与业务码规范与全局错误约定一致。

---

## 5. 非功能需求

| 项 | 要求 |
|----|------|
| 安全 | HTTPS 部署；敏感操作二次确认；权限校验以服务端为准 |
| 性能 | 权限数据可缓存；用户权限变更后缓存失效或版本号失效 |
| 可用性 | 管理后台关键操作有明确成功/失败反馈 |
| 可维护性 | 权限标识命名规范文档化，与前端路由 name、后端注解一致 |

---

## 6. 验收标准（摘要）

- 创建用户 A（仅角色 R1），R1 仅含菜单 M1；登录后仅见 M1，访问其他路由被拒绝。
- 调整 R1 去掉 M1 后，用户 A 在缓存策略约定时间内不再能访问 M1（刷新或重新登录后必生效）。
- 未携带合法凭证调用受保护接口 → 401；凭证合法但无权限 → 403。
- 管理员对用户、角色、权限的增删改在审计日志中可查（若本期包含审计）。
