# AI Burst 界面操作手册

本文面向**日常在浏览器里操作**的用户，按**页面与按钮**说明怎么用当前系统。权限、部署与接口细节见根目录 [README.md](../README.md)；MAG 业务背景补充见 [MAG功能使用指南.md](./MAG功能使用指南.md)。

---

## 1. 登录与整体布局

### 1.1 登录

1. 浏览器打开系统地址（本地开发一般为 `http://localhost:5173`）。
2. 在登录页输入 **用户名**、**密码**，点击 **「登录」**。
3. 登录成功后进入主界面；若失败，页面或顶部会有错误提示。

页面底部提示：首次环境可能使用 **admin / admin123**（以实际部署为准）。

### 1.2 主界面三块区域

| 区域 | 位置 | 作用 |
|------|------|------|
| 侧栏菜单 | 左侧深色条 | 点击进入各功能页；**没有某项菜单 = 当前账号无权限** |
| 顶栏 | 上方 | 显示当前用户昵称/用户名；**「退出」** 登出 |
| 内容区 | 中间 | 当前菜单对应的列表、表单、工作台等 |

**说明**：菜单来自后端按角色下发，**不是**所有人相同。升级权限后请 **重新登录** 再查看菜单。

### 1.3 无权限时

- 访问无权的地址可能进入 **403** 页面。
- 许多按钮带权限控制：**看不到按钮** 通常表示无该项操作权限，需管理员在 **角色管理** 里分配权限。

## 2. 系统管理

侧栏展开 **「系统管理」**（名称以你环境为准），常见子菜单如下。

### 2.1 用户管理

- 表格浏览用户列表。
- **新增 / 编辑 / 删除 / 重置密码** 等按钮在有权时显示；按弹窗或表单填写后 **保存**。

### 2.2 角色管理

- 维护角色列表。
- **分配权限**：勾选菜单与按钮级权限，保存后，绑定该角色的用户需 **重新登录** 生效。

### 2.3 菜单管理

- 维护系统菜单树（路径、组件、排序等），一般由管理员使用。

## 3. 大模型

### 3.1 通道配置

**路径**：侧栏 **「大模型」→「通道配置」**（或名称相近）。

| 操作 | 步骤 |
|------|------|
| 查看列表 | 进入页面即加载表格（ID、名称、厂商、协议、Base URL、默认模型、状态等） |
| 新增通道 | 点 **「新增通道」** → 弹窗中选 **厂商**、填 **通道名称**、**Base URL**、**API Key**（新建必填）、可选 **默认模型** 等 → **「保存」** |
| 编辑 | 行内 **「编辑」** → 修改后保存；**API Key 可留空表示不修改** |
| 删除 | 行内 **「删除」**，确认后删除 |
| 启用/禁用 | 在新增/编辑弹窗中用 **状态** 开关（启用 / 禁用） |

### 3.2 对话体验

**路径**：**「大模型」→「对话体验」**。

1. **通道**：下拉框 **选择已配置的通道**（须先完成「通道配置」且通道为启用状态）。
2. **覆盖模型**：可选；不填则使用通道上的默认模型。
3. 在底部输入框输入内容，**Ctrl+Enter** 或点 **「发送」**。
4. **「清空会话」**：清除当前对话展示（有消息时可用）。
5. 若返回用量信息，可展开 **「本次用量（JSON）」** 查看。

未选通道就发送时，会提示先选择通道。

## 4. 多 Agent 协作（MAG）

侧栏 **「多Agent协作」** 下常见菜单与路由对应关系如下（与数据库种子一致，名称以界面为准）。

| 菜单 | 大致作用 |
|------|----------|
| **MAG 项目** | 项目列表；进入 **工作台** |
| **MAG 大屏** | 选项目、看快照统计；可显示 WebSocket 连接状态 |
| **MAG 知识库** | 知识条目查询、新建、编辑、删除 |

---

### 4.1 MAG 项目列表

1. 点击 **「MAG 项目」**。
2. 表格展示项目：ID、名称、Agent 数、需求版本、最近活动、状态等。
3. **「工作台」**：进入该项目的 **项目工作台**（核心操作区）。
4. **「新建项目」**（有权限时）：弹出对话框填 **名称** → **「创建」**。
5. **「归档」**（有权限时）：按提示确认后归档项目。
6. 底部分页：**上一页 / 下一页** 或点页码换页。

---

### 4.2 MAG 大屏

1. **「项目」** 下拉框选择项目（加载自项目列表）。
2. **「刷新」**：重新拉取大屏快照数据（如任务按状态统计）。
3. **「WS」** 标签：显示 WebSocket **已连接 / 未连接**（与实时推送相关）。

---

### 4.3 MAG 知识库

| 操作 | 步骤 |
|------|------|
| 查询 | **关键词** 输入后回车或点 **「查询」** |
| 新建 | **「新建」** → 弹窗填 **标题**、**正文**（必填）、**关键词**（可选）→ **「保存」** |
| 编辑 | 行 **「编辑」** → 修改 → **「保存」** |
| 删除 | 行 **「删除」** → 确认 |
| 翻页 | 表格底部分页 |

---

### 4.4 项目工作台（最重要）

从 **MAG 项目** 列表点击某一行的 **「工作台」** 进入。页面上方为 **项目标题与状态**；下方为 **标签页**，按标签切换功能（**无权限时部分按钮不显示**）。

| 标签 | 常见操作 |
|------|----------|
| **成员** | 查看项目成员表格（用户 ID、用户名、昵称、角色） |
| **Agent** | **「新建 Agent」** 选类型、填名称保存；表格行 **「触发 run」** 提交编排（成功/失败见顶部 **消息**：绿色成功、橙色警告） |
| **模块** | **「新建模块」**、**「导入蓝图」**（从 **知识库 KB 条目 ID** 复制占位模块）、**「刷新」**；行 **「编辑」/「删除」** |
| **任务** | **「新建任务」**；行 **「开始」**、**「申报完成」**（手动；亦可能由系统在编排成功且编排有非空最终回复、且配置开启时自动标为已完成）、**「阻塞」**、**「要活」**（随任务状态出现） |
| **需求文档** | 大文本框编辑；**「保存新版本」**；**「版本列表」**；版本号 **对比**；**「变更影响分析」** |
| **沟通** | **「新线程」**；行 **「消息」** 打开对话；**「触发编排」** |
| **编排执行** | **「刷新」**；查看各次编排状态与结果摘要 |
| **告警** | **「刷新」**；未确认行 **「确认」** |
| **定时任务** | 有权限才显示；**「新建/更新」**、**「刷新（本项目）」** |

**线程消息弹窗**：输入内容发送后列表刷新；在沟通标签内操作。

**触发 run / 触发编排 的提示**：

- **绿色成功**：通常表示请求被接受；若配置了 Temporal 且正常，文案中可能包含 **workflowId**。
- **橙色警告**：如未启用 Temporal、连不上服务、或业务拒绝等，请阅读完整提示文案。

## 5. 操作习惯与提示

1. **表格**：表头排序、列宽若支持以界面为准；行内 **蓝色链接** 多为「编辑 / 查看 / 工作台」等。
2. **对话框**：底部 **「取消」** 关闭不保存；**「保存」/「确定」** 提交。
3. **分页**：改页后数据重新加载；每页条数若可改，在分页器上操作。
4. **顶部消息（Element Plus）**：成功、警告、错误多为简短说明；复杂错误可打开浏览器 **开发者工具 → 网络** 查看接口返回。
5. **登录态过期**：可能自动跳转登录页，重新登录即可。
